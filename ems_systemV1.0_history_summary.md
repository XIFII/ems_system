# ems_systemV1.0 历史对话与迭代总结

## 1. 文档目的

本文件用于沉淀本项目在当前会话中的关键需求、设计决策、页面重构过程和最终实现状态，便于后续继续迭代时快速回溯上下文，而不需要重新翻阅完整对话。

---

## 2. 项目背景

- 项目类型：园区级微电网边缘 EMS 前端 Demo
- 技术形态：单页前端项目，当前以本地演示态数据为主
- 产品定位：纯边缘 EMS，不保留 Cloud/Edge 双架构切换入口
- 当前重点模块：
  - 策略中心
  - 电价配置
  - L1 硬性约束
  - 动环策略配置
  - L3 智能与调度 / 削峰填谷

---

## 3. 整体产品方向决策

### 3.1 纯边缘 EMS

- 删除顶部 `Edge / Cloud` 架构切换
- 策略中心固定为边缘 EMS 视图
- 当前保留页签：
  - `模式切换 (L2)`
  - `硬性约束 (L1)`
  - `动环策略配置`
  - `智能与调度 (L3)`

### 3.2 配置安全优先

多个页面从“直接可编辑”改为“默认只读 + 显式编辑 + 保存/取消”的安全交互，避免误操作：

- DI/DO 点位配置
- 月份模板分配
- 日期覆盖分配
- 动环策略编辑

### 3.3 单一生效模式原则

电价配置采用“当前仅存在一种生效电价模式”的产品规则：

- 要么固定分时电价
- 要么动态市场化分时电价
- 不再作为两套并行配置长期共存

这一原则也同步影响：

- 电价配置页展示逻辑
- L3 模板曲线背景展示逻辑
- 智能规划策略的数据来源

## 4. 动环策略配置模块演进

### 4.1 初始需求来源

用户提供：

- `中耀达-动环逻辑表6月14日更新.xls`
- 两张表格截图
  - IO 点位表
  - 动环逻辑表

### 4.2 关键产品化决策

为解决一个 EMS 下挂多个 DI/DO 采集器、每个采集器内部都可能存在 `DI1 ~ DIn / DO1 ~ DOn` 重名的问题，确定采用：

- `采集器设备 + 点位编码` 作为唯一标识

即：

- 不允许仅按裸 `DI1/DO1` 配置
- 策略中引用点位时必须落到具体设备

### 4.3 页面结构调整

一级页签 `动环策略配置` 下拆成二级页签：

- `DI/DO点位配置`
- `动环策略配置`

### 4.4 DI/DO 页面最终规则

- 只负责给物模型点位命名
- 不支持新增
- 不支持删除
- 不支持切换所属采集器字段
- 不配置触发类型、优先级、启用状态
- 未使用点位通过命名为 `预留` 处理
- 默认只读
- 通过整页 `编辑 / 保存 / 取消` 维护

### 4.5 关联采集设备弹窗

弹窗结构确定为：

- 左侧：候选采集器列表
- 右侧：当前选中采集器的点位预览

支持：

- 多设备关联
- 点位预览只读
- 当前阶段仅前端演示，不接真实接口

### 4.6 动环策略页面最终规则

- 列表默认只读
- 支持：
  - 新增
  - 编辑
  - 复制
  - 启用/停用
- 不支持删除
- 新增/编辑统一采用居中大弹窗
- 不再使用底部展开编辑区

### 4.7 策略弹窗设计

弹窗分区：

- 基础信息区
- 判断条件区
- 执行动作区
- 底部保存区

#### 判断条件支持

- 设备选择
- 点位选择弹窗
- 判断规则
- 判断依据

#### 执行动作支持

- 设备动作
- 平台动作
- 前移/后移排序
- 预留拖拽排序语义

## 5. L1 硬性约束模块演进

### 5.1 页面定位

L1 只保留两类策略：

- 防逆功率保护
- 需量控制

### 5.2 交互重构

从“页面堆叠详细配置”重构为：

- 页面展示摘要卡
- 每张卡独立开关
- 每张卡打开自己的配置弹窗

### 5.3 弹窗拆分

用户明确要求：

- 防逆流和需量控制不能共用同一个弹窗

最终实现为两个完全独立弹窗：

- `showAntiBackflowModal`
- `showDemandControlModal`

### 5.4 当前字段

两类弹窗均按现场常用简化版收敛为：

- 限制值
- 目标值
- 保护程度
- 说明
- 保存

## 6. L3 智能与调度 / 削峰填谷模块演进

### 6.1 页面布局演进

经历过多轮调整，最终确定：

- 上方：策略分配区
- 下方：模板工作区

原因：

- 策略分配属于高频操作
- 模板维护属于低频操作

### 6.2 模板与分配的边界

最终明确：

- 模板工作区负责定义模板
- 策略分配区负责把模板应用到月份/日期
- 主预览始终跟随当前选中模板
- 日期覆盖不再接管主图表预览

### 6.3 月/日分配交互演进

#### 月份分配

- 默认只读
- 通过 `编辑 / 保存 / 取消` 进入安全编辑态
- 修改月模板后，保存时会清除该月所有日覆盖

#### 日期覆盖

- 默认只读
- 通过 `编辑 / 保存 / 取消` 进入安全编辑态
- 单日只允许下拉选择现有模板
- 不允许从日期格直接编辑模板内容

### 6.4 批量复用逻辑

最终确定：

- 批量复用和多日选择全部放到弹窗里完成
- 主页面不再提供：
  - 日期勾选
  - 全选当月
  - 清空选择
  - 清除已选日覆盖

弹窗内保留：

- 模板选择
- 多日选择
- 全选当月
- 清空
- 应用到已选 N 天

### 6.5 日期视图

削峰填谷日期区采用真实日历排版：

- 周一到周日
- 节假日标签
- 默认/覆盖状态标签

### 6.6 模板编辑弹窗

最终能力：

- 新建模板
- 编辑模板
- 复制模板
- 时段增删
- 动作与功率正负自动联动
- 保存前校验

### 6.7 智能规划策略

在模板编辑弹窗中恢复并保留：

- `智能规划策略` 按钮

并实现为二级子窗口，支持：

- 选择规划日期
- 模拟计算
- 结果审核
- 应用到当前模板

### 6.8 智能规划与电价联动

智能规划结果不再是固定演示内容，而是跟随当前电价模式：

- 固定模式：读取固定电价模板
- 动态模式：读取当前动态市场化电价数据

应用规则：

- 结果整批覆盖模板时段
- 不追加、不局部合并

### 6.9 模板曲线视觉

后续多轮调整后，最终确定：

- 功率采用折线/阶梯折线表达
- 固定分时模式：显示 TOU 色带
- 动态模式：显示市场电价辅助底带 + 价格折线
- 动态模式下不展示尖峰平谷分层卡片

## 7. 电价配置模块演进

### 7.1 初始方向

最早实现为：

- 固定分时电价
- 动态市场化分时电价
- 通过 tab 形式切换

### 7.2 关键产品决策

用户明确指出：

- 企业侧只能同时存在一种电价机制
- 不会同时启用固定和动态两套价格

最终重构为：

- `电价模式设置`
- 当前唯一生效模式：
  - 固定分时电价
  - 动态市场化分时电价

### 7.3 固定分时电价

保留：

- 尖 / 峰 / 平 / 谷 / 深谷
- 模板名称
- 适用范围
- 模板库
- 时段编辑
- 色带预览

### 7.4 动态市场化分时电价

最终规则：

- 支持 `48点 / 30分钟`
- 支持 `96点 / 15分钟`
- 每日对应一条价格曲线
- 支持：
  - 手工表格录入
  - CSV/Excel 导入
  - 保存状态展示

### 7.5 动态电价语义收敛

用户明确要求：

- 动态市场化分时电价不应再映射尖峰平谷等级

最终实现为：

- 动态电价仅保留原始市场价格
- 删除动态等级列
- 删除动态等级分层逻辑
- 删除动态模式下的尖峰平谷文案
- 图表底部只保留统一的市场价辅助底带

### 7.6 动态电价日历视图

新增按日历查看的日电价区域，支持：

- 周一到周日
- 节假日
- 每日平均电价
- 点击日期切换当前编辑日
- 上月 / 下月切换

### 7.7 动态电价数据存储规则

前端演示态已按：

- 日期
- 粒度

进行本地状态缓存。

即：

- 同一日期、同一粒度的数据可独立维护
- 切日期时会加载对应日期的数据
- 导入和保存只影响当前日期、当前粒度
